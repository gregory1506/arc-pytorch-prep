Module 06 - FastAPI Deployment

Clean API design for segmentation inference.

Theory highlights

  • Separate endpoints for health, prediction, and metadata.
  • Validate requests with Pydantic and return clear errors.
  • Use lazy model loading to avoid slow startup.
  • Use async only for I/O bound work.

Flashcards

Q: What should /health return?
A: A simple status payload and HTTP 200 to confirm liveness.
Q: When do you use async endpoints?
A: For I/O bound tasks like file reads or network calls.
Q: Why not load the model at import time?
A: It slows startup and makes testing harder.
Q: How do you limit risk from large images?
A: Enforce content type and size limits before processing.
Q: Why use Pydantic models?
A: They validate input types and generate documentation automatically.
