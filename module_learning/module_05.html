Module 05 - ONNX Export

Export, verify, and benchmark for deployment.

Theory highlights

  • ONNX is a framework-agnostic graph format for deployment.
  • Always export in eval mode to avoid stochastic layers.
  • Verify outputs within a tolerance to ensure correctness.
  • Quantization improves CPU speed at a possible accuracy cost.

Flashcards

Q: Why is ONNX useful in production?
A: It decouples inference from PyTorch and enables optimized runtimes.
Q: Why must the model be in eval mode for export?
A: To freeze dropout and batch norm behavior for deterministic output.
Q: What does dynamic axes enable?
A: Variable batch size or spatial dimensions at inference.
Q: How do you verify export correctness?
A: Compare ONNX output to PyTorch output with a small tolerance.
Q: What is a common ONNX export pitfall?
A: Using an input shape that does not match the model.
